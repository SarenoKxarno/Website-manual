Структура формы

<form>

Все формы помещены в тег <form> кроме этого всегда должен быть указан атрибут <action.

action

Атрибут action содержит в себе значение URL-адрес страницы получателя введённых данных при отправке формы на сервер.

method

Формы можно отправлять при помощи одного из двух методов get или post. Выглядеть это будет примерно так: "method="get"
Метод get создан для:

  • Коротких форм(например поисковый запрос)
  • При простом получении данных с Web-сервера(без отправки сведений в базу данных)
Метод post подойдёт если форма:
  • Позволяет пользователям загружать файлы на сайт
  • Очень длинные
  • Содержат конфиденциальные данные
  • Добавляет или удаляет сведения или удаляет их из базы данных
Если атрибут method не указан то форма будет отправлена на сервер с помощью метода get
Ввод текста

maxlength

Атрибут maxlength можно использовать для ограничения количества символов, которое можно ввести пользователю в данном поле.

id

id используется для различения формы и других элементов разметки страницы.

size

Важно! Не использовать атрибут size ранее его могли использовать для управления шириной поля ввода данных. В настоящий момент, размер поля редактируеться исключительно в файлах формата CSS.

<input>

Элемент <input> используется для создания компонентов формы нескольких типов. Значение атрибута type указывает этот тип.

type="text"

Когда артибут type указывает значение text это значит что будет созданно однострочное поле ввода текста.

name

При заполненении и отправке формы с данными, сервер должен знать, какие сведения относяться к какому элементу формы. Достигается это при помощи атрибута name который присваевает формам класс, к которым после происходит обращение.

Атрибут name устанавливает имя поля ввода пороля, отправляемого на сервер вместе с поролем, введённым пользователем.

Текстовая область

<textarea>

Элемент <textarea> используется для создания многострочных облостей ввода текста.

Работы над визуальным составляющим сайта проводяться в файлах типа CSS, но на старых сайтах можно увидеть такие атрибуты как cols и rows.
cols устанавливает ширину текстовой области, которая измеряется в количестве символов, а rows задаёт высоту поля в строках.

Переключатели

Атрибут type="radio" позволяет посетителям сайта, выбрать только один вариант из предложенных.

Атрибут name отправляется серверу вместе с значением выбранного переключателя. Если ответ на вопрос подразумевает выбор одного из нескольких вариантов, все переключатели формирующие список ответов должны иметь одинаковое значение атрибута name.

Атрибут checked указывает на то, какой из предлагаемых вариантов должен быть выбран по умолчанию при загрузки страницы(если это необходимо). Атрибут принемает единственное значение - checked. Данный атрибут должен быть установлен только у одного переключателя из группы.

type="checkbox"

Раскрывающийся список

<select>

Для создания раскрывающегося списка, используют элемент <select> он может содержать два и более элементов <option>.

name

Атрибут name опрежеляет имя элемента формы, отправляемое на сервер вместе с выбранным значением.

<option>

Элемент <option> используеться для указания вариантов ответа, которые может выбрать посетитель сайта.

value

Атрибут value в элементе <option> используеться для установки значения отправляемого на сервер вместе с именем выбранного элемента.

selected

Атрибут selected устанавливает вариант ответа, который должен быть показан по умолчанию при загрузке страницы. Если этот элемент отсутствует, то будет показан первый вариант ответа. Функционал списка аналогичен переключателям(выбран может быть только один вариант).
Однако у каждого элемента есть своё преймущество:

  • Если необходимо чтобы посетитель сайта видел все варианты ответа сразу, то использовать лучше переключатели.
  • Если список варианто вдостаточно длинный, то лучше подойдёт раскрывающийся список.

Добавив атрибут multiple со значением multiple можно позволить посетителям сайта выбрать сразу несколько вариантов ответа.

Метки элементов формы

Группировка элементов формы

Злементы формы связанные по смыслу, можно сгруппировать в fieldset это бывает особенно полезно при создании длинных форм.
Большинство брузеров отобразят fieldset в виде рамки вокруг элементов формы. Внешний вид рамки модет быть изменён с помощью CSS.
Эдемент legend содержащий надпись обозначения группы элементов формы, как правило, помещают сразу после открывающего тега элемента fieldset.

Валидация данных формы

В старых версиях HTML использовали сценарии написанные на языке JavaScreept но в HTML5 появилась собственная технология валидации.
Примером встренной системы проверки может быть атрибут required.
