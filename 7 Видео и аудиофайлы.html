Теги видеоплеера

<video>

  • Элемент <video> обладает большим набором атрибутов позволяющих управлять воспроизведением ролика.
  • Элемент <poster> создаёт превью на видео, до того как оно начнёт вопсроизведение.
  • Атрибут src позволяет указать место положение нужного файла.
  • Атрибуты width и height определяют размеры окна проигрывателя.
  • Атрибут controls нужен чтобы бразуер предоставил собственные элементы воспроизведения.
  • Атрибут autoplay нужен чтобы виоролик сам включился после загрузки воспроизведение видеоролика.
  • Атрибут loop используеться чтобы зациклить воспроизведение видеоролика.

preload

Атрибут preload даёт инструкцию браузеру, что нужно делать при загрузки страницы. У него может быть одно из следующих значений:

  1. none - брузер не должен начинать загрузку ролика, пока на него не щёлкнули.
  2. auto - браузер должен загружать видеоролик, по окончанию загрузки страницы.
  3. metadata - бразуер должен просто собрать метаинформацию(размер, первый кадр, список воспроизведения и длительность)

Добавление аудиоконтента

<audio>

В HTML5 присутсвует элемент <audio> он создан чтобы добавлять на старницу элементы аудио файлов.
Для этого элемента существует несколько атрибутов.

Атрибуты элемента <audio>:

  • Атрибут src определяет путь к нужному аудиофайлу.
  • Атрибут controls указывает нужно ли отображать элементы управления прогрывателя. По умолчанию, при его отсутствии элементы отображаться не будут. Но при помощи JavaScript можно настроить отображение собственных элементов управления.
  • Атрибут autoplay указывает, что проигрывание записи должно начинаться атвтоматически.
  • Атрибут preload данный атриубт указывает какие действия должен выполнять браузер, если для проигрывателя не указан атриубт "autoplay".
  • Атрибут loop указывает, что аудиофайл должен быть зациклен.
